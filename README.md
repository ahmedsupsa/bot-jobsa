# بوت تليجرام - التقديم على الوظائف

بوت لتسجيل المستخدمين بكود تفعيل، حفظ بياناتهم وسيرهم الذاتية، وربط Gmail لإرسال التقديمات من إيميلهم.

## المتطلبات

- Python 3.10+
- حساب Supabase (قاعدة بيانات)
- بوت تليجرام من [@BotFather](https://t.me/BotFather)

## الإعداد

1. **استنساخ/فتح المشروع**
   ```bash
   cd "ملجد بوت تليجرام"
   ```

2. **إنشاء بيئة افتراضية وتثبيت الحزم**
   ```bash
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **إعداد Supabase**
   - ادخل إلى [Supabase Dashboard](https://supabase.com/dashboard) → مشروعك.
   - من **SQL Editor** شغّل محتوى الملف `database/schema.sql` لإنشاء الجداول.
   - من **Settings → API** انسخ:
     - Project URL → `SUPABASE_URL`
     - anon/public key أو Service Role key → `SUPABASE_KEY`

4. **ملف البيئة `.env`**
   - انسخ `.env.example` إلى `.env`
   - املأ القيم:
     ```
     BOT_TOKEN=رقم_البوت_من_BotFather
     SUPABASE_URL=https://xxx.supabase.co
     SUPABASE_KEY=المفتاح_الذي_نسخته
     ```

5. **رموز التفعيل**
   - أضف رموز تفعيل من Supabase في جدول `activation_codes`:
     - `code`: النص الذي سيدخله المستخدم
     - `subscription_days`: عدد أيام الاشتراك (مثلاً 30)

6. **تخزين السير الذاتية (Supabase Storage)**
   - أنشئ **Bucket** باسم `cvs`: Supabase → **Storage** → **New bucket** → الاسم: `cvs`
   - **مهم:** حتى يظهر الملف في الـ bucket بعد الرفع من البوت، شغّل سياسات التخزين من **SQL Editor**:
     - افتح الملف `database/storage_policies.sql` ونفّذ محتواه (سياسة INSERT و SELECT لـ bucket `cvs`).
   - إن كان جدول `user_cvs` موجوداً مسبقاً: `ALTER TABLE user_cvs ADD COLUMN IF NOT EXISTS storage_path TEXT;`

## تشغيل البوت

```bash
python main.py
```

بشكل افتراضي يعمل البوت بوضع **Polling** (يسحب التحديثات من تليجرام).

### (اختياري) تشغيل بوضع Webhook (خادم ويب)

إذا كان لديك عنوان **HTTPS** عام (نطاق أو نفق مثل ngrok / Cursor Ports)، يمكنك تشغيل البوت كخادم ويب لاستقبال التحديثات مباشرة:

1. في `.env` أضف أو عدّل:
   ```
   USE_WEBHOOK=true
   WEBHOOK_URL=https://عنوانك-العام.com
   WEBHOOK_PORT=8080
   WEBHOOK_PATH=webhook
   ```
2. تأكد أن المنفذ `WEBHOOK_PORT` مفتوح وأن الوصول من الإنترنت يصل إلى هذا المنفذ (أو استخدم تحويل منفذ / reverse proxy).
3. شغّل البوت: `python main.py`. سيسجّل تليجرام عنوان الويب تلقائياً ويُرسل التحديثات إليه.

**ملاحظة:** تليجرام يتطلب HTTPS. في التطوير المحلي يمكن استخدام [ngrok](https://ngrok.com) أو ميزة Ports في Cursor للحصول على رابط HTTPS مؤقت.

## أدمن من نفس البوت (بدون بوت ثاني ولا لوحة ويب)

1. **حدد نفسك كأدمن** في `.env`:
   ```
   ADMIN_TELEGRAM_IDS=رقم_تليجرام_حسابك
   ```
   احصل على رقمك من [@userinfobot](https://t.me/userinfobot) في تليجرام (الرقم يظهر كـ **Id**).

2. في البوت اكتب الأمر:
   ```
   /admin
   ```
   ستظهر لك قائمة:
   - **توليد أكواد تفعيل**: أرسل سطراً بصيغة `49 365` (عدد الأكواد ثم أيام الاشتراك).
   - **إضافة وظيفة**: يطلب منك عنوان الوظيفة، الوصف، الشركة، رابط التقديم (يمكن تخطي بكتابة -).
   - **إضافة إعلان**: يطلب عنوان الإعلان ثم النص.

لا تحتاج بوت ثاني ولا تشغيل لوحة ويب؛ كل الإدارة من نفس البوت.

(اختياري) **لوحة ويب**: إذا أردت إدارة من المتصفح شغّل `python -m admin.app` وافتح http://127.0.0.1:5000 بعد إضافة `ADMIN_PASSWORD` و`ADMIN_SECRET` في `.env`.

## توليد أكواد تفعيل من الطرفية

```bash
python scripts/add_activation_codes.py -n 49 -d 365
```
(مثال: 49 كود، اشتراك 365 يوم = سنة)

## مسار المستخدم (ملخص)

1. **Start** → اختيار: "مشترك لدي كود تفعيل" أو "ارغب بالاشتراك".
2. **كود التفعيل** → إدخال الكود → في حال الموافقة: تهنئة + "تسجيل مستخدم".
3. **التسجيل** → اسم، جوال، عمر، مدينة → حفظ مع كود الاشتراك.
4. **القائمة الرئيسية**: التقديمات، حسابي، الإعدادات.
5. **التقديمات**: عدد المرسلة، سجل التقديمات، تفضيلات الوظائف (مجالات عامة/خاصة + بحث).
6. **حسابي**: بياناتي، السيرة الذاتية (رفع/معاينة)، حالة الاشتراك.
7. **الإعدادات**: ربط Gmail + كلمة مرور التطبيق، قوالب التقديم (رسمي/عادي/احترافي) مع معاينة وإرسال للإيميل، لغة التقديم (عربي/إنجليزي مع تحذير بعدم التغيير حتى انتهاء الاشتراك)، تواصل معنا.

## ملاحظات

- **كلمة مرور التطبيق (Gmail)**:
  1. من المتصفح افتح صفحة حساب Google وسجّل الدخول بنفس إيميل Gmail الذي ستستخدمه في البوت.
  2. من قسم "الأمان" فعّل **التحقق بخطوتين** (إن لم يكن مفعّلاً).
  3. بعد التفعيل ادخل إلى صفحة **كلمات مرور التطبيقات** واختر التطبيق "البريد" والجهاز "أخرى".
  4. انسخ كلمة المرور ذات 16 خانة التي تظهر لك واستخدمها في البوت بدلاً من كلمة مرور الحساب العادية.
  5. رابط مباشر لصفحة كلمات مرور التطبيقات: https://myaccount.google.com/apppasswords
- **قاعدة البيانات**: استخدم المفتاح المناسب (anon للعميل، service role للإدارة) حسب سياسة المشروع.
