## 2. نظام عجلة الحظ 🎡

**الفكرة:** كل تقديم ناجح يرسله البوت → نقرة عجلة → جائزة مالية عشوائية

**قاعدة البيانات (جداول جديدة):**
- `spin_rewards` — يسجل كل نتيجة عجلة (المبلغ، الحالة، التاريخ)
- `reward_withdrawals` — طلبات سحب المكافآت
- عمودان جديدان في `users`: `pending_spins` (نقرات انتظار) + `reward_balance` (الرصيد المتراكم)
- دالة RPC: `increment_pending_spins(user_id)` — تزيد النقرات بأمان

**الـ Worker (Edge Function):**
- بعد كل تقديم ناجح → يستدعي `increment_pending_spins` تلقائياً

**APIs:**
- `GET /api/portal/rewards` — جلب الرصيد + النقرات + السجل
- `POST /api/portal/rewards/spin` — تشغيل العجلة (تنقص نقرة، تُضاف جائزة)
- `POST /api/portal/rewards/withdraw` — طلب سحب (الحد الأدنى 30 ريال)

**الجوائز:**
| الجائزة | الاحتمال |
|---|---|
| 5 هللات | 25% |
| 10 هللات | 22% |
| 25 هللات | 18% |
| 50 هللة | 14% |
| 1 ريال | 10% |
| 1.5 ريال | 5% |
| 2 ريال | 4% |
| 5 ريالات | 1.5% |
| 10 ريالات (بعد 180 يوم فقط) | 0.5% |

**صفحة المستخدم `/portal/rewards`:**
- عجلة متحركة بألوان وأقسام
- رصيد + نقرات متبقية
- شريط تقدم نحو 30 ريال
- سحب بـ 3 طرق: تحويل بنكي / محفظة إلكترونية / رصيد اشتراك
- سجل كل النتائج والسحوبات

**التنقل:** رابط "عجلة الحظ 🎡" أُضيف للقائمة الجانبية والسفلية

**ملف migration:** `database/migrations/spin_rewards.sql`

## 4. إصلاح استخراج المسميات من السيرة الذاتية

**المشكلة الأصلية:** "لم يتمكن الذكاء الاصطناعي من استخراج المسميات"

**الأسباب:**
1. ترتيب خاطئ: الـ prompt يُرسل قبل الملف (النماذج تعمل أفضل بالعكس)
2. لم يُحدد نوع الاستجابة كـ JSON
3. مهلة قصيرة (30 ثانية) لا تكفي لتحليل PDFs
4. أسماء نماذج خاطئة أو منتهية الصلاحية

**الإصلاح:**
- الملف يُرسل أولاً ثم التعليمات
- `responseMimeType: "application/json"` يجبر Gemini على إرجاع JSON
- المهلة رُفعت لـ 55 ثانية
- النماذج المستخدمة (مُثبتة من الـ Worker): `gemini-2.5-flash`, `gemini-2.5-flash-lite`, `gemini-2.0-flash`, `gemini-2.0-flash-lite`
- محلل JSON مرن + fallback نصي إذا فشل JSON
